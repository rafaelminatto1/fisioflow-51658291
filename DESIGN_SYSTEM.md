DESIGN SYSTEM - FISIOFLOW

Visão Geral
- Conjunto de tokens, diretrizes de componentes e padrões visuais usados pela Web e Apps móveis para manter consistência de marca e experiência do usuário.

Tokens de Design
- Cor: paleta primária/secundária; modo claro/escuro; variantes dinâmicas (quando disponíveis).
- Tipografia: fontes, pesos e escalas.
- Formas: border radius, radii e grid spacing.
- Espaçamento, tamanho de fontes, tamanhos de botões, gaps.

Formato e Implementação
- Tokens podem ser exportados como JSON/TS e consumidos por componentes.
- Mapeamento de tokens para CSS-in-JS (ou CSS vars) dependendo da stack atual.
- Convenção de nomes e organização de pastas para tokens e components.

Guia de Componentes
- Padrão de componentes comuns (Button, Input, Card, Dialog, List, Avatar, etc.).
- Regras de acessibilidade (ARIA), estados (hover, focus, disabled) e responsive design.
- Como compor novas peças usando o Design System existente.

Conformidade e Governança
- Processos de revisão de novos tokens/Componentes.
- Documentação de mudanças no DESIGN_SYSTEM.md sempre que tokens ou componentes forem alterados.
- Governança de versões do design system (quando aplicável).

Exemplos de Uso
- Demonstrações de aplicação de tokens em componentes simples (cor de primary button, spacing de cards).

Notas
- Este documento deve evoluir com a evolução da UI; mantenha-o atualizado.
